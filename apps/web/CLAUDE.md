# Web UI (apps/web)

Last verified: 2026-05-16

Next.js 16 App Router with route group. Routes: `/` (redirects to `/notebooks`), `/notebooks`, `/notebooks/[id]`, `/notebooks/[id]/wiki`, `/notebooks/[id]/wiki/[slug]`, `/conversations`, `/ask`, `/documents`, `/workflows`, `/demo` (everything except `/` lives inside the `(app)` route group). The legacy `/evals` route is no longer exposed in the sidebar — the `Evaluations` nav entry was replaced by `Demo`.

**Route Group Convention:** All app routes except `/` live in `app/(app)/`. The `(app)` folder is a Next.js route group — it groups routes under a shared layout (`app/(app)/layout.tsx` with `AppSidebar`) without changing the URLs. So `/ask` resolves at `app/(app)/ask/page.tsx`, `/notebooks/[id]` at `app/(app)/notebooks/[id]/page.tsx`, etc.

**Landing:** `app/page.tsx` is a server component that calls `redirect("/notebooks")` — the notebooks grid is the primary entry point. The older feature-card landing is gone.

Key files:
- `components/AppSidebar.tsx` — persistent left sidebar. `NAV_ITEMS` order: Notebooks (top), Ask, Conversations, Documents, Workflows, Demo. Rendered by `app/(app)/layout.tsx`. Uses `usePathname` to highlight the active route (matches exact href or any subpath via `startsWith`). Hidden on mobile (`hidden md:flex`).
- `components/NotebookGrid.tsx` — main `/notebooks` view: fetches `GET /notebooks`, renders a grid of `NotebookCard`s, owns the create-notebook form. Empty state prompts new users to create their first notebook.
- `components/NotebookCard.tsx` — individual notebook tile. **Whole card is navigable** via an invisible overlaid `<Link>` (`absolute inset-0 z-0`, `tabIndex={-1}`, `aria-label={notebook.name}`); card content sits on `relative z-10` and the visible "Open" button on `relative z-20` so all three remain clickable. Action buttons (delete / cancel / arm-delete) call `e.preventDefault() + e.stopPropagation()` inside their `onClick` so clicking them doesn't also trigger the card-level Link. Renders `notebook.document_count` when the field is present (singular/plural pluralization via `=== 1`).
- `components/NotebookHeader.tsx` — title strip rendered at the top of `/notebooks/[id]` and `/notebooks/[id]/wiki/*` showing notebook name + description and entry points to the wiki and conversation views.
- `components/WikiGenerateButton.tsx` — kicks off `POST /notebooks/{id}/wiki/generate`, polls `GET /notebooks/{id}/wiki/jobs/{job_id}` for status (`queued | running | completed | failed`), and shows `pages_done / pages_total` progress. Disables itself while a job is in flight.
- `components/WikiTreeNav.tsx` — left-rail navigation for `/notebooks/[id]/wiki/*` showing `structure.sections` and the page slugs inside each section. The "current" page is highlighted via `usePathname`.
- `components/WikiPage.tsx` — renders one wiki page's markdown content. Mounts `MermaidDiagram` for fenced ```` ```mermaid ```` blocks via a `react-markdown` `components` override. Also renders **source document citation badges** below the content (`sourceDocIds` prop → named badges linking to `${API}/documents/{id}/source`, with `ExternalLink` icon, falling back to short ID when name is unavailable). Custom `a` renderer in `markdownComponents` intercepts `href?.startsWith('/notebooks/')` and uses Next.js `<Link>` for SPA wiki cross-page navigation; all other links get `target="_blank" rel="noopener noreferrer"`.
- `components/MermaidDiagram.tsx` — client-only Mermaid renderer (dynamic import of `mermaid`). Falls back to a code block if rendering fails. The Mermaid syntax conventions are enforced in `wiki_prompts._MERMAID_RULES` on the backend so diagrams stay parseable.
- `components/ConversationList.tsx` / `components/ConversationItem.tsx` — pure presentational components for the `/conversations` history page. `ConversationItem` displays title, source notebook (resolved via the page's `notebookMap`), and timestamp. The page itself owns fetching `GET /conversations` and the optional `?notebook_id=` filter.
- `components/AnswerCard.tsx` — renders Answer (markdown via react-markdown), Citations, **Figures** (image grid), and Retrieved Chunks panels in that order. Wraps content in `CitationProvider` so `CitationBadge` children can read citation data via context. Citations are clickable when the API returns a `document_id` — the link is `${NEXT_PUBLIC_API_URL}/documents/<id>/source#page=<page>` via `sourceHref(documentId, page)`. When an `onSelectCitation` callback is passed (from ConversationView), clicking a citation opens the SourcePanel instead. Figure `<img>` `src` is built as `${NEXT_PUBLIC_API_URL}${fig.image_url}` because `image_url` from the API is server-relative (`/figures/...`). Answer text is rendered with `prose prose-sm max-w-none` classes and the `remarkCitationBadges` plugin active so `[N]` markers become interactive `CitationBadge` components. **Copy-answer button** is rendered in the header (`ml-auto`) unconditionally — outside the `needs_human_review` conditional — so users can always copy the answer text regardless of confidence label; click flips the icon to a check for 2 s via `setTimeout`.
- `components/CitationBadge.tsx` — inline `[N]` citation chip rendered inside answer prose. Reads `citations` and `retrievedContexts` from `CitationContext` by 1-based index. Shows a `HoverCard` on hover (openDelay 150 ms) with the chunk's verbatim text, page, and document name. Clickable (calls `onSelectCitation`) only when `document_id` and `page` are present; otherwise renders as a greyed, non-interactive chip. Out-of-bounds index renders the fallback chip.
- `components/CitationContext.tsx` — React context (`CitationProvider` / `useCitationContext`) that passes `citations[]`, `retrievedContexts[]`, and optional `onSelectCitation` callback down to `CitationBadge` children without prop-drilling. Wraps `AnswerCard` content and the streaming partial view in `ConversationView`.
- `components/ChatInput.tsx` — text input with a `#`-triggered document picker (combobox + ARIA listbox). Typing `#` anywhere in the field opens a dropdown of up to 8 ingested documents (filtered by substring). Arrow/Enter/Escape navigate and select; selecting a doc attaches it as a badge chip and removes the `#token` from the text. Submit calls `onSubmit(text, attached?)`. Props: `pending`, `documents: SourceDocument[]`, `onSubmit`.
- `components/QueryModeToggle.tsx` — segmented `Docs | General` control (role="group") wired to `useQueryMode`. `Docs` mode sets `doc_only: true` on the next message; `General` mode leaves it unset. Persisted in `localStorage` across page reloads via `useQueryMode`.
- `components/ConversationView.tsx` — main ask interface; owns conversation history (messages state), streaming hook (useConversationStream), `queryMode` (useQueryMode), `documents` (fetched from `GET /documents` on mount for the picker), and `selectedCitation` state for source panel. Accepts optional props `notebookId?: string` and `documentIds?: string[]` so it can be embedded inside a notebook page (e.g. `app/(app)/notebooks/[id]/page.tsx`) and scope every message to the notebook's R2R collection; when `notebookId` is set the title renders as an `h2` so the page heading can stay an `h1`. Renders `SuggestedQuestions` on empty state. Renders `QueryModeToggle` and `ChatInput` in the footer. If `ChatInput` provides no attached documents, the prop-passed `documentIds` are used instead. The streaming partial view wraps content in `CitationProvider` + `remarkCitationBadges` so in-flight `[N]` markers render as greyed badges while streaming. Lazy-loads SourcePanel via next/dynamic (ssr: false) so pdfjs is deferred until a citation is clicked.
- `components/SourcePanel.tsx` — shadcn Sheet that opens when a citation is clicked. Loads chunks manifest from GET /documents/{id}/chunks, renders the source PDF at the cited page via react-pdf, and overlays a yellow bbox highlight on the cited chunk (border + 50% opacity fill via `bg-yellow-300/50`). The overlay is **suppressed when the chunk's bbox covers ≥90% of both page dimensions** (`isFullPageBbox` from `lib/bboxConversion.ts`) — those are body-column "full page" chunks where a highlight would obscure rather than highlight the cited text. A **cited-passage text strip** is rendered below the PDF viewer (`overlayChunk?.text_preview`, `line-clamp-4`) so the verbatim chunk text is visible even when the bbox isn't drawn. Lazy-loaded; configured in lib/pdfWorker.ts (CDN worker URL — requires network; for offline see RISKS section in phase_07.md).
- `lib/types.ts` — shared TypeScript types: `Citation` (includes optional `document_id` + `page_end` + `chunk_id` + `chunk_index` for deep linking), `RetrievedContext` (includes optional `chunk_id` for CitationBadge context lookup), `FigureAsset` (`figure_id`, `source_file`, `page_number`, `image_url`, optional `caption`), the `AskResponse` envelope (always includes `figures: FigureAsset[]` and optional `doc_only_not_found?: boolean`), `WorkflowResponse` (mirrors `SupportState` from `packages/workflows/`), `IngestJob` (`status: queued | running | completed | failed`), `SourceDocument` (`document_id`, `source_file`, `source_url`, `size_bytes`, `updated_at`), `Notebook` (`id`, `name`, `description`, `r2r_collection_id`, timestamps, optional `document_count?: number` populated by the `GET /notebooks` LEFT JOIN), and `NotebookDocument` (`notebook_id`, `document_id`, `added_at`).
- `lib/remarkCitationBadges.ts` — remark plugin (`# pattern: Functional Core`). Replaces `[N]` **and comma-separated groups `[N, M, K]`** (spaces optional — `[2,4,5]` and `[2, 4, 5]` both match) in the markdown AST with one `citationRef` node per number; each becomes a `<cite-ref data-num="N">` hast element. Used by `AnswerCard` (committed answers) and the streaming partial view in `ConversationView`. Ignores `code` and `inlineCode` blocks. Consumer components must map `cite-ref` → `CitationBadge` in their `react-markdown` `components` prop.
- `lib/bboxConversion.ts` — pdfplumber/PyMuPDF → CSS-pixel coordinate helpers (`# pattern: Functional Core`). Exports `bboxToCssBox(bbox, dims)` (scales `[x0, top, x1, bottom]` PDF points to rendered CSS pixels using `pageWidthPx/naturalWidthPt` ratios), `findOverlayChunk(chunks, citation, pageNum)` (resolves the cited chunk by `chunk_index` first, falling back to page-range overlap with a bbox), and `isFullPageBbox(bbox, naturalWidthPt, naturalHeightPt)` (true when both axes ≥ 0.9 of page dimensions — used by `SourcePanel` to suppress overlays that would cover the whole page).
- `lib/docTitleFormatter.ts` — `formatDocTitle(sourceFile)` (`# pattern: Functional Core`). Strips the trailing file extension, replaces `_` / `-` with spaces, collapses whitespace, trims, then truncates to 40 visible chars with a single `…` (no trailing space before the ellipsis). Used by `SuggestedQuestions` for doc-titled chip seeds (`Summarize <title>`); pure helper so other surfaces can adopt the same canonical display form later.
- `components/SuggestedQuestions.tsx` — empty-state question chips rendered by `ConversationView`. On mount fetches `GET /documents` (first 2 docs → `Summarize <formatDocTitle(source_file)>` seeds) and then `GET /documents/{firstDoc.document_id}/questions` for cache-first LLM-generated questions. Falls back to three `STATIC_SEEDS` when the API has no documents or `questions` returns empty. Final list is `Array.from(new Set([...docSuggestions, ...seeds])).slice(0, 6)` — deduplicated and capped at 6. Shows skeletons while the `/questions` fetch is in flight; both fetches share an `AbortController` cleaned up on unmount.
- `lib/useQueryMode.ts` — hook (`# pattern: Imperative Shell`). Returns `[mode, setMode]` where `mode: "documents" | "general"`. Persists to `localStorage` under key `docquery.queryMode`; default is `"documents"`. SSR-safe (reads localStorage in `useEffect` only).
- `app/(app)/layout.tsx` — route group layout that wraps all app pages with AppSidebar + main pane flexbox. Provides the persistent sidebar navigation.
- `app/(app)/ask/page.tsx` — one-shot Q&A page that wraps `ConversationView`; conversation state is component-local. Renders a SourcePanel (lazy-loaded) that opens when citations are clicked, showing the source PDF with bbox highlights.
- `app/(app)/documents/page.tsx` — async upload via `POST /ingest/jobs` then polling `GET /ingest/jobs/{id}`; lists existing source PDFs from `GET /documents` (includes a `/chunks` route for retrieving chunk metadata used by SourcePanel) and supports `DELETE /documents/{id}` with **inline two-step confirmation** (first click arms the row, second click within the same row commits — clicking another row or the same row's Cancel resets). The poller uses an `AbortController` that fires on unmount and on a fresh upload to prevent `setState` after unmount. Quality report path is still nested under `result.quality_report` (commit f45a5ec); `data.quality_report` is the wrong shape. Uses shadcn `Table`, `Progress`, and `Button` components.
- `app/(app)/workflows/page.tsx` — single page with a mode toggle (`triage` ↔ `email`) that posts to `/workflows/support/triage` or `/workflows/support/email-draft`. Renders the `WorkflowResponse` with an "Approval required" / "Ready to send" badge derived from `requires_human_approval`. The older `/workflows/support` sub-route was replaced by this top-level page.
- `app/page.tsx` — server component (outside the route group, no sidebar). Calls `redirect('/notebooks')` — the feature-card landing was removed; the notebooks grid is the primary entry point.
- `app/(app)/demo/page.tsx` — portfolio showcase page composing four panels: `WikiPreviewPanel`, `QAShowcasePanel`, an inline figure section (plain `<img>` to `/demo/example_figure.png`), and `DemoAskBox`. Server component except for `DemoAskBox` (client). Designed to render entirely from snapshot data when the API/notebook is unavailable so the page is always demoable.
- `app/(app)/demo/WikiPreviewPanel.tsx` — server component. Reads `./data/wiki_structure.json` and renders `WikiTreeNav` + `WikiPage` with the first page's content. JSON shape: `{title, sections: [{title, pages: [{slug, title}]}], first_page_content?: string}` — `sections` is structurally checked against the `WikiStructure` type at compile time (no `as` cast). `setup_demo_corpus.py` transforms the backend's `{page_slugs: string[]}` shape into the frontend's `{pages: [{slug, title}]}` shape before writing the snapshot.
- `app/(app)/demo/QAShowcasePanel.tsx` — server component. Reads `./data/example_qa.json` (`AskResponse` shape) and renders an `AnswerCard`. The JSON IS the cached `/ask` response — `setup_demo_corpus.py` writes it verbatim.
- `app/(app)/demo/DemoAskBox.tsx` — client component (`"use client"`, Imperative Shell). Reads `NEXT_PUBLIC_DEMO_NOTEBOOK_ID` and `NEXT_PUBLIC_API_URL`. Pings `${API}/health` with `AbortSignal.timeout(2000)` on mount; on 2xx renders `<ConversationView notebookId={...} />`; on miss/timeout/empty notebook id, renders an `AnswerCard` from cached `example_qa.json` with a "Live demo unavailable — showing cached example" `Badge`. Three states: `loading` (skeleton), `live` (real conversation), `offline` (cached). React 19 silently no-ops the post-unmount setState so no AbortController is wired through.
- `app/(app)/demo/data/wiki_structure.json` / `example_qa.json` — committed snapshot data. Real content is generated by `scripts/setup_demo_corpus.py` and overwrites the placeholders. Both are intentionally tracked in git so the `/demo` page renders out-of-the-box on a fresh clone.
- `public/demo/example_figure.png` — committed figure snapshot. Placeholder is a 1×1 PNG (~70 bytes); `setup_demo_corpus.py` replaces it with the first extracted figure from the ingested corpus. The setup script's `_figure_is_real()` check uses a 1024-byte threshold to detect the placeholder.

## Stream / query contracts

`useConversationStream.sendMessage(text, opts?)` accepts `opts.docOnly?: boolean`, `opts.documentIds?: string[]` (plural; renamed during the notebooks rollout), and `opts.notebookId?: string`. These map onto the POST body to `/conversations/{id}/messages/stream` as `doc_only`, `document_ids`, and `notebook_id`. When `opts.notebookId` is provided and no conversation exists yet, the hook also includes `notebook_id` + `first_message` (the trimmed user text) in the `POST /conversations` body so the conversation history is filed under that notebook and the title is seeded. The API uses `doc_only` for the post-hoc strict not-found check (`_DOC_ONLY_NOT_FOUND`), `document_ids` to scope retrieval to specific documents, and `notebook_id` to resolve an `r2r_collection_id` server-side (collection-scoped retrieval takes precedence over document-id filtering).

`AskResponse.doc_only_not_found?: boolean` — truthy when the API replaced the answer with the not-found string. `AnswerCard` shows a "Switch to General" CTA when this flag is set, calling its `onSwitchToGeneral` prop.
