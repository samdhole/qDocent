## Prerequisites

- Fresh clone or `data/notebooks.db`, `data/wiki.db`, `data/conversations.db` removed for a clean baseline.
- `.env` populated (`GOOGLE_API_KEY`, `R2R_BASE_URL=http://localhost:7272`, `CORS_ORIGINS=http://localhost:3000`).
- Three terminals running in parallel:
  - `make r2r` (port 7272 — `r2r_gemini.toml` present)
  - `make api` (port 8000)
  - `make web` (port 3000)
- `make test` is green locally (mirrors CI).
- A small fixture PDF on disk (e.g. `data/sample_docs/company_policy.pdf` from `python scripts/create_sample_docs.py`).
- A non-PDF file (e.g. any `.exe` or `.txt` file) for rejection testing.

## End-to-End: Notebook → Ingest → Wiki → Conversation

Purpose: Validates AC1 + AC2 + AC3 + AC4 in one chained flow that mirrors the primary user journey.

1. Create a fresh notebook "E2E NB" via the UI.
2. Upload two PDFs to the notebook (use `python scripts/create_sample_docs.py` to get 3 sample PDFs).
3. Click "Generate Wiki" and wait for completion (auto-redirect to first slug).
4. Navigate the WikiTreeNav, opening at least two pages — confirm sidebar highlight tracks current page.
5. On one wiki page, click "Ask about this page" and ask a question grounded in that page's content; confirm citations match `source_doc_ids` for that page.
6. Navigate to `/conversations` — the conversation from step 5 appears, filed under "E2E NB", with a truncated title.
7. Delete the wiki (`curl -X DELETE .../wiki`), then visit `/notebooks/<id>/wiki` — empty state renders; generation can be retriggered.
8. Delete the notebook from `/notebooks` — card disappears; `/documents` still lists both uploaded PDFs.

## End-to-End: Restart survives migration (AC5.1, AC5.2)

Purpose: Confirms the startup hook wiring on a real filesystem (unit tests cover logic; this confirms lifespan integration).

1. Stop the API. Delete `data/notebooks.db`, `data/wiki.db`, `data/conversations.db`.
2. Confirm `data/documents/<doc-id>/manifest.json` files exist for at least one previously-ingested document.
3. Run `uvicorn apps.api.main:app --reload --port 8000`. Inspect logs — startup completes without raising.
4. `sqlite3 data/notebooks.db "SELECT name FROM notebooks; SELECT COUNT(*) FROM notebook_documents;"` → exactly one "Default Notebook" + non-zero member count matching the manifest count.
5. Stop and restart the API. Re-run the same queries — counts unchanged (no duplicates) (AC5.2).
