## End-to-End: Restart-during-ingest

1. Delete `data/ingest_jobs.db`. Start `make r2r`, `make api`, `make web`.
2. Upload a large PDF via the UI. Note the `job_id` from the network tab.
3. While job is `running`, kill the API process (Ctrl+C).
4. Restart `make api`.
5. Refresh the UI documents page or curl `/ingest/jobs/<job_id>`.

**Expected:** Job shown as `failed` with `error="interrupted by restart"`. Re-fetching within a minute still returns the failed row (TTL window fresh). UI surfaces the failure cleanly without hanging.

---

## End-to-End: 404 boundaries

1. `curl -i http://localhost:8000/ingest/jobs/this-id-does-not-exist` → HTTP 404.
2. Manually expire a completed job: `sqlite3 data/ingest_jobs.db "UPDATE jobs SET updated_at = datetime('now', '-2 hours') WHERE status = 'completed' LIMIT 1;"` (capture its `job_id` first).
3. `curl -i http://localhost:8000/ingest/jobs/<that_job_id>` → HTTP 404.
4. Confirm deletion: `sqlite3 data/ingest_jobs.db "SELECT job_id FROM jobs WHERE job_id = '<that_job_id>';"` → no rows.
